# 产品需求文档

## 目标与用户

本系统供分析管理员查看多个 AI Pixel 账户的请求用量与余额流水。服务每分钟采集数据到 SQLite，提供全账户汇总和单账户下钻。

## v1 功能

- config.toml 配置多个账户、Go 本机监听端口、分析时区和上游节点优先级。
- 上游凭据和网页登录固定键 DASHBOARD_USERNAME、DASHBOARD_PASSWORD 位于 .env。
- 网页凭据缺失时安全随机生成并以 0600 权限原子写回；SQLite 仅保存 Argon2id 带盐哈希。
- 初始化同步上月第一天至当前时间，之后每分钟增量同步。
- 默认小时聚合，可切换日粒度，并分别对比昨天同小时和上周同小时。
- 分析请求数、Token、成本、延迟、模型、API Key、端点、请求和计费类型。
- 余额入账、出账和净额分离；原始流水类型、业务分类、方向、备注和引用字段可筛选。
- 展示账户节点、同步时间、数据新鲜度、错误摘要与手动重试。
- 网页会话固定 24 小时。

## 非功能要求

Linux 产物为内嵌前端的 CGO-free 单一可执行文件。Nginx 是唯一公网 HTTPS 入口，Go 默认监听 127.0.0.1 的配置端口。单账户失败不影响其他账户，秘密不得进入日志或 SQLite 明文字段。界面桌面优先且移动端可用。

## 不包含

v1 不提供网页编辑凭据、多管理员权限、外部 AI 备注分类、分布式部署、外部数据库或自动清理历史数据。

## 验收

多账户同步、汇总隔离、两种同期、余额方向、流水类型与备注筛选、登录保护、24 小时会话、Nginx 代理和 Linux 单二进制均须通过自动化或浏览器验证。
