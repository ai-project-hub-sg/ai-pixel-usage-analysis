# UI 设计

## 导航

登录后包含总览、请求分析、余额流水和账户状态。顶部提供账户、时间范围、小时/日粒度、最近更新时间和退出。

## 页面

- 登录：用户名、密码、提交状态和统一错误；成功建立 24 小时 HttpOnly Cookie。
- 总览：请求、Token、实际成本、入账、出账、净额 KPI；当前、昨天同期、上周同期使用不同线型并可独立开关。
- 请求：按账户、API Key、模型、入口/上游端点、请求、流式和计费类型筛选，展示成本、Token、延迟、缓存、排行和明细。
- 流水：入账、出账、净额分离；按原始类型、业务分类、方向、备注关键词、引用类型/ID 筛选；下钻标准化备注和 metadata。
- 账户：当前节点、最近成功、数据新鲜度、错误摘要和手动重试；部分失败显示非阻塞警告。

## 交互与无障碍

桌面侧栏，窄屏单列。控件至少 44px，具备标签、可见焦点和非颜色提示。每个区域独立处理骨架、空数据、部分失败、错误和重试；会话失效返回登录页。
