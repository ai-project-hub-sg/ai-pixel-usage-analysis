# ai-pixel-usage-analysis
.env中有登录的账号和密码。config.toml中配置了域名组和权重。1为最优先，99为最劣后。如果权重相同。则按顺序依次测试。要求从/usage获取余额流水和请求记录。初始化时获取本月和上月的余额流水。后续每分钟从上游请求一次数据，增量更新进入本地sqlite数据库。对余额流水和请求记录形成分析。默认页面展示每小时的聚合结果，有同环比对比。其中余额流水要把入方向和出方向分开。其他需要能筛选的，你补充完善分析页面。需要一个前端来展示分析结果。go语言做后端。需要生成一个二进制的包，无其他依赖，可以直接部署在liunx服务器上。先做prd、ui、trd、结构文档、开发规范、验收方案、打包文档。然后再开始实现。后续会接入多个账户。需要能提供全账户汇总和查看单账户的能力。
